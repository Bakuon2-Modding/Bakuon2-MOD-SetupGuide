## 事前準備（バックアップ）

> **重要**: MODを導入する前に、必ず以下のバックアップを取ってください。

### 1. ゲームデータのバックアップ

ゲームフォルダ全体をコピーしてバックアップしてください。

```
bakuon2/               ← このフォルダをコピー
├── bakuon2.exe
├── bakuon2_Data/
├── MonoBleedingEdge/
└── ...
```

### 2. セーブデータのバックアップ

セーブデータは以下の場所に保存されています。エクスプローラのアドレスバーに貼り付けてアクセスし、フォルダごとバックアップしてください。

```
%USERPROFILE%\AppData\LocalLow\bakumatsu\bakuon2
```

### 3. アセットバンドルデータのバックアップ

> **最重要**: このデータは必ずバックアップしてください。

アセットバンドルデータとは、ゲームの3Dモデル、シーン、音声などを含む1GB超の大容量データです。ゲーム起動時にサーバーからダウンロードされ、ローカルに保存される仕組みでした。

**サービスが終了した現在、このデータを再取得する方法はありません。** このデータがないとゲームの3Dモデルやシーンが表示されません。もしゲームのトップ画面にあった「ダウンロードデータ削除」ボタンを押してしまっていた場合、残念ながらデータは喪失しており、復旧は困難です。

アセットバンドルデータは以下の場所に保存されています。`bakumatsu_bakuon2/` フォルダを探し、フォルダごとバックアップしてください。

```
%USERPROFILE%\AppData\LocalLow\Unity
```

## MOD環境構築

### ゲームフォルダのコピー

> **強く推奨**: MODはオリジナルのゲームフォルダには導入せず、コピーしたフォルダに導入してください。

オリジナルのゲームフォルダを直接改変してしまうと、MODが原因で問題が起きたときに元に戻すのが大変です。以下の手順でMOD専用のゲームフォルダを用意しましょう。

1. 事前準備でバックアップしたゲームフォルダをコピーする
2. コピーしたフォルダの名前を `bakuon2_modded` などに変更する
3. 以降の作業はすべてコピーしたフォルダに対して行う

```
D:\Games\bakuon2\             ← オリジナル（改変しない）
D:\Games\bakuon2_modded\      ← MOD用（こちらに導入する）
```


### BepInEx のセットアップ

BepInEx は Unity 製ゲームにMOD（プラグイン）を読み込ませるためのオープンソースのフレームワークです。

#### インストール手順

1. [BepInEx 5.4.23.5](https://github.com/BepInEx/BepInEx/releases/tag/v5.4.23.5) から `BepInEx_win_x64_5.4.23.5.zip` をダウンロード
2. ZIPの中身をMOD用ゲームフォルダに展開
3. 一度 `bakuon2.exe` を起動して終了（BepInExの初期設定が生成されます）

#### フォルダ構成

セットアップ完了後のフォルダ構成は以下のようになります。

```
bakuon2_modded/
├── bakuon2.exe
├── bakuon2_Data/
├── winhttp.dll              ← BepInEx ローダー
├── doorstop_config.ini      ← BepInEx 設定
└── BepInEx/
    ├── core/                ← BepInEx 本体
    ├── plugins/             ← MODを配置するフォルダ
    └── config/              ← MOD設定ファイル
```

### アセットバンドルの配置

MOD用ゲームフォルダ内に `dl_Data/` フォルダを作成し、バックアップしておいたアセットバンドルデータ (`bakumatsu_bakuon2/`) をコピーしてください。


```
bakuon2_modded/
├── bakuon2.exe
├── BepInEx/
└── dl_Data/
    └── bakumatsu_bakuon2/   ← バックアップからコピー
        ├── scene/
        ├── city/
        ├── suteage/
        ├── accessory/
        ├── animation/
        ├── audio/
        ├── roomeditor/
        └── sprite/
```

本来のゲームが参照するデータと、MODが参照するデータを明確に分離するために再配置します。元の場所（`AppData\LocalLow\Unity`）のデータはそのまま残しておいてください。

> **注意**: アセットバンドルの配置場所はMODによって異なる場合があります。各MODのドキュメントを確認してください。

## MODの導入方法

1. 導入したいMODのDLLファイルを入手
2. `BepInEx/plugins/` フォルダにDLLファイルをコピー
3. ゲームを起動

MODが正しく読み込まれたかどうかは、ログファイル (`BepInEx/LogOutput.log`) で確認できます。

## アンインストール方法

### BepInEx を完全に削除する場合

以下のファイル・フォルダを削除すると元の状態に戻ります。

| 削除対象 | 説明 |
|---------|------|
| `winhttp.dll` | BepInEx ローダー |
| `doorstop_config.ini` | BepInEx 設定 |
| `BepInEx/` | BepInEx 本体・プラグイン |

### 特定のMODのみ削除する場合

`BepInEx/plugins/` フォルダから該当するDLLファイルを削除してください。

## トラブルシューティング

### BepInEx が動作しない

以下を確認してください。

1. `winhttp.dll` と `doorstop_config.ini` が `bakuon2.exe` と同じフォルダにあるか
2. ログファイル (`BepInEx/LogOutput.log`) が存在するか
   - 存在しない場合、BepInExが正しくインストールされていません

### MOD が読み込まれない

以下を確認してください。

1. DLLファイルが `BepInEx/plugins/` フォルダに配置されているか
2. ログファイル (`BepInEx/LogOutput.log`) にエラーが出力されていないか
3. MOD環境構築手順を正しく実行したか。BepInEx のセットアップとアセットバンドルのコピーの手順を入れ替えた場合、MODが機能しない場合があります

### ⚠重要

環境によっては、デスクトップにゲームを配置しているとMOD環境（BepInEx）が動作しない場合があります。別の場所に移動してトライしてみてください。
